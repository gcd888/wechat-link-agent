## 1. 安全架构概述

微连作为本地桌面应用，安全模型以 **本地数据保护** 和 **敏感信息加密** 为核心，不涉及服务端安全。

### 1.1 安全设计原则

- **本地优先**：用户数据仅存储在本地，除非主动开启 WebDAV 同步
- **加密存储**：API Key 等敏感信息使用 AES-256-GCM 加密后存储
- **最小权限**：仅请求必要的系统权限，不收集用户数据
- **密钥隔离**：加密密钥仅存内存，应用退出清除

### 1.2 安全层级

```
┌─────────────────────────────────────────┐
│  应用层安全                              │
│  - 主密码保护（PBKDF2 派生）             │
│  - API Key 加密（AES-256-GCM）           │
│  - 设备信任缓存（可选）                   │
├─────────────────────────────────────────┤
│  传输层安全                              │
│  - WebDAV HTTPS 加密传输                 │
│  - iLink Bot API HTTPS                   │
├─────────────────────────────────────────┤
│  存储层安全                              │
│  - SQLite 本地文件                       │
│  - safeStorage 系统级加密（bot_token）    │
│  - 敏感字段加密存储                       │
├─────────────────────────────────────────┤
│  进程层安全                              │
│  - Electron contextBridge 隔离           │
│  - 子进程超时控制（5 分钟）               │
│  - IPC 通道命名隔离                       │
└─────────────────────────────────────────┘
```

## 2. STRIDE 威胁分析

### 2.1 威胁矩阵

| 威胁类型 | 威胁描述 | 风险等级 | 缓解措施 |
|----------|----------|----------|----------|
| **欺骗 (Spoofing)** | 伪造微信身份发送指令 | 中 | iLink Bot API token 认证 |
| **篡改 (Tampering)** | 篡改 Agent CLI 输出 | 低 | 子进程 stdout 直接读取，无中间层 |
| **篡改 (Tampering)** | 篡改数据库文件 | 中 | 文件系统权限 + 加密敏感字段 |
| **否认 (Repudiation)** | 用户否认发送过消息 | 低 | 消息持久化记录（含 source 字段） |
| **信息泄露 (Info Disclosure)** | API Key 泄露 | 高 | AES-256-GCM 加密存储 |
| **信息泄露 (Info Disclosure)** | bot_token 泄露 | 高 | Electron safeStorage 加密 |
| **拒绝服务 (DoS)** | Agent 子进程无限运行 | 中 | 5 分钟超时自动终止 |
| **拒绝服务 (DoS)** | 微信长轮询阻塞 | 低 | 指数退避重连 |
| **权限提升 (Elevation)** | IPC 通道越权调用 | 低 | contextBridge 隔离 + 通道命名规范 |

### 2.2 攻击面分析

| 攻击面 | 入口点 | 防护措施 |
|--------|--------|----------|
| 微信消息 | iLink Bot API 长轮询 | 命令路由白名单、消息长度限制 |
| Agent 子进程 | CLI 参数注入 | 参数化传递、工作目录限制 |
| WebDAV 同步 | 远程文件覆盖 | 下载前自动备份 |
| 数据库文件 | 本地文件直接读取 | 敏感字段加密、文件系统权限 |
| IPC 通道 | 渲染进程注入 | contextBridge 隔离、无 Node.js 集成 |

## 3. 密钥管理与加密

### 3.1 加密密钥链

```
用户输入主密码
      │
      ▼
PBKDF2(password, salt, 100000, 32, sha256)
      │
      ▼
32 字节加密密钥（仅存内存）
      │
      ├──▶ AES-256-GCM 加密 API Key
      │    ├── 随机 IV (16 bytes)
      │    ├── 加密密文
      │    └── Auth Tag
      │
      └──▶ 设备信任缓存（可选）
           └── ~/.wechat-link-agent/.trust
               （存储派生密钥，免重复输入）
```

### 3.2 加密参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 算法 | AES-256-GCM | 认证加密，同时保证机密性和完整性 |
| 密钥派生 | PBKDF2 | 从主密码派生密钥 |
| 迭代次数 | 100,000 | 防暴力破解 |
| 哈希算法 | SHA-256 | PBKDF2 的哈希函数 |
| 密钥长度 | 32 字节 (256 位) | AES-256 所需 |
| IV 长度 | 16 字节 (128 位) | GCM 模式标准 |
| Salt | 固定值 | `wechat-link-agent-salt-v1` |

### 3.3 加密对象

| 加密对象 | 加密方式 | 存储位置 | 说明 |
|----------|----------|----------|------|
| API Key | AES-256-GCM | llm_providers 表 | 密文 + IV + Tag 三字段存储 |
| bot_token | Electron safeStorage | accounts 表 | 系统级加密（ENC1: 前缀） |
| WebDAV 密码 | Electron safeStorage | sync_config 表 | 系统级加密（ENC1: 前缀） |

### 3.4 主密码生命周期

| 阶段 | 操作 | 说明 |
|------|------|------|
| 设置 | `masterPassword:set` | 首次设置，派生密钥存内存 |
| 解锁 | `masterPassword:unlock` | 每次启动输入密码，派生密钥存内存 |
| 设备信任 | `masterPassword:unlock(trustDevice=true)` | 缓存密钥到 .trust 文件 |
| 恢复 | `masterPassword:tryRestore` | 从 .trust 文件恢复密钥 |
| 锁定 | `masterPassword:lock` | 清除内存中的密钥 |
| 修改 | `masterPassword:change` | 旧密码验证 → 新密码派生 → 重新加密所有数据 |
| 清除 | `masterPassword:clear` | 删除 .trust 文件 + 清空所有加密数据 |

## 4. 访问控制

### 4.1 主密码保护

| 操作 | 是否需要解锁 | 说明 |
|------|-------------|------|
| 查看 Agent 列表 | 否 | 公开信息 |
| 发送消息 | 否 | 不涉及加密数据 |
| 查看 API Key | 是 | 需要解密 |
| 修改 API Key | 是 | 需要加密新值 |
| 修改主密码 | 是 | 需要验证旧密码 |
| WebDAV 配置 | 否 | safeStorage 自动处理 |

### 4.2 IPC 权限

所有 IPC 通道通过 `contextBridge` 暴露，渲染进程无法直接访问 Node.js API。IPC 通道按模块命名空间隔离，无通配符通道。

## 5. 数据安全

### 5.1 数据隔离

| 隔离维度 | 实现方式 |
|----------|----------|
| 会话隔离 | 每个会话独立消息记录（session_id 外键） |
| Agent 隔离 | 每个 Agent 独立配置和会话列表 |
| 来源隔离 | 消息标记来源（desktop/wechat） |
| 用户隔离 | 单用户应用，无多租户 |

### 5.2 数据生命周期

| 数据类型 | 创建 | 存储 | 删除 |
|----------|------|------|------|
| 会话消息 | 用户发送消息时 | messages 表 | 删除会话级联删除 |
| Agent 配置 | PATH 扫描时 | agents 表 | 重新扫描时清空重建 |
| API Key | 用户配置时 | llm_providers 表（加密） | 删除供应商时删除 |
| 微信凭证 | 扫码登录时 | accounts 表（加密） | 断开连接时删除 |
| 备份文件 | 手动/同步前 | 本地文件 | 手动删除 |

### 5.3 日志安全

- 日志不记录 API Key、主密码等敏感信息
- 日志文件存储在 `~/.wechat-link-agent/logs/`
- 日志格式：`app-YYYY-MM-DD.log`
- 不自动上传日志到服务器

## 6. 已知安全隐患

以下为评估发现的非阻塞隐患，建议后续整改：

| 编号 | 隐患描述 | 风险等级 | 当前状态 | 建议整改方案 |
|------|---------|---------|---------|------------|
| H-01 | WebDAV 密码使用 safeStorage 加密，但部分 Linux 发行版 safeStorage 不可用 | 低 | 已缓解 | 降级时提示用户 |
| H-02 | 设备信任文件（.trust）存储派生密钥，物理访问可提取 | 低 | 用户可选 | 已通过"可选"降低风险 |
| H-03 | 长消息拆分时可能截断 Markdown 语法 | 低 | 已处理 | 按段落拆分 |

## 7. 安全检查清单

发布前安全检查：

- [ ] API Key 在数据库中为密文存储
- [ ] bot_token 使用 safeStorage 加密
- [ ] 日志中无敏感信息
- [ ] IPC 通道无通配符
- [ ] 子进程有超时控制
- [ ] WebDAV 密码加密存储
- [ ] 主密码修改后所有加密数据重新加密
- [ ] contextBridge 未暴露 Node.js API
